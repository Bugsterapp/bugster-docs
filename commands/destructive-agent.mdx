## Overview

**Purpose**: Runs AI-powered destructive agents to find potential bugs by simulating edge cases and unexpected user behavior.

**When to use**:
- **Before major releases** to catch edge case bugs
- On staging environments to stress-test changes
- After significant code modifications
- When you want to find vulnerabilities proactively

**Important notes**:
- **Only targets pages with recent code changes** (git diff)
- Agents like "UI Crashers" and "Form Destroyer" attempt to break your app
- **Can cause data corruption** - use on test/staging environments only
- Records videos of destructive attempts for debugging
- More aggressive than regular testing

## CLI Reference

| Command | Description | Example |
|---------|-------------|---------|
| `bugster destructive` | Run destructive agents on all changed pages | `bugster destructive` |
| `bugster destructive --headless` | Run without browser UI | `bugster destructive --headless` |
| `bugster destructive --parallel 5` | Run up to 5 agents concurrently | `bugster destructive --parallel 5` |
| `bugster destructive --limit 8` | Limit number of agents to run | `bugster destructive --limit 8` |

### Execution Flags

| Flag | Description | Example |
|------|-------------|---------|
| `--headless` | Run agents in headless mode (no browser UI) | `--headless` |
| `--silent` | Run agents silently (minimal output) | `--silent` |
| `--verbose` | Show detailed agent execution logs | `--verbose` |
| `--max-concurrent` | Maximum number of concurrent agents (default: 3) | `--max-concurrent 6` |
| `--limit` | Maximum number of destructive agents to run (default: 5, min: 1) | `--limit 10` |
| `--browser` | Browser to use for destructive testing | `--browser firefox` |

### Output Flags

| Flag | Description | Example |
|------|-------------|---------|
| `--stream-results` | Stream destructive results as they complete (default: true) | `--stream-results` |
| `--no-stream-results` | Disable streaming to dashboard | `--no-stream-results` |
| `--base-url` | Override base URL from config | `--base-url https://staging.myapp.com` |

### Agent Types & Priority

| Agent Type | Priority | Description | Example Behaviors |
|------------|----------|-------------|-------------------|
| **UI Crashers** | Highest | Attempts to crash UI components | Rapid clicking, invalid inputs, boundary testing |
| **Form Destroyer** | High | Targets forms with malicious inputs | SQL injection attempts, XSS payloads, buffer overflows |
| **Navigation Chaos** | Medium | Tests navigation edge cases | Back/forward spam, deep linking, invalid routes |

### Common Usage Patterns

| Use Case | Command | Description |
|----------|---------|-------------|
| Pre-release testing | `bugster destructive --limit 15` | Comprehensive destructive testing |
| Quick vulnerability scan | `bugster destructive --limit 3 --headless` | Fast security check |
| CI/CD integration | `bugster destructive --headless --stream-results` | Automated destructive testing |
| Staging validation | `bugster destructive --base-url https://staging.app.com` | Test staging environment |
| Performance testing | `bugster destructive --parallel 8` | Maximum concurrent destruction |

**Safety Notes**:
- **Never run against production** - destructive agents may cause real damage
- Use test databases and isolated environments
- Agents may submit forms, delete data, or trigger irreversible actions
- Review video recordings to understand what caused any detected issues

**Pro Tips**:
- Run after `bugster update` to test newly modified pages
- Higher `--limit` values increase chance of finding edge case bugs
- Videos are automatically saved for any bugs found
- Best used as final validation step before releases
